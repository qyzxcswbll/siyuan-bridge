## 二、sy-doc —— 文档管理

### 适用场景

用户说「把这段内容保存到思源」「读取某篇文档」「删除某个笔记」「把文档 A 移动到文件夹 B 下面」「查看笔记本 C 的文档结构」「更新某篇文档的内容」「在文档末尾追加一段话」。

### Action: create

创建新文档。对应原 Python 版 `sy-save`。

**参数：**

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `content` | string | ✅ | — | Markdown 内容 |
| `notebook` | string | 否 | 索引 0 | 笔记本序号或名称 |
| `title` | string | 否 | 从 content 提取 | 文档标题 |

**行为：**

1. content 不能为空，否则返回错误
2. notebook 为空时使用默认笔记本（mapper 解析）
3. title 为空时自动从 content 提取第一个 `# ` 或 `## ` 标题行
4. 自动生成唯一路径 `/{safe_title}-{timestamp}`，避免思源路径去重机制
5. 如果 content 中匹配到已配置的 codebase repo 名称，自动归类到 `/projects/{name}/` 下
6. 调用 Siyuan API `createDocWithMd` 写入
7. 返回文档 ID 和标题

**返回示例：**

```
✅ 已保存
- 📂 知识库/我的笔记
- 🔗 siyuan://blocks/20210101000000-abc1234
```

**边界：**
- content 为空 → ❌ 内容不能为空
- notebook 映射失败 → 自动刷新笔记本列表重试一次
- 思源未运行 → ❌ 思源笔记未运行
- API 返回错误 → ❌ 保存失败：{msg}

---

### Action: read

读取文档内容。对应原 Python 版 `sy-read`。

**参数：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | string | ✅ | 文档 ID（`siyuan://blocks/xxx` 中的 xxx） |

**行为：**

1. id 不能为空
2. 调用 `getDoc` API 获取文档
3. 返回文档的 Markdown 内容

**返回：** 文档的 Markdown 原文

**边界：**
- id 为空 → ❌ 文档 ID 不能为空
- 文档不存在（API 返回空或错误）→ ❌ 读取失败：{msg}
- 文档内容为空 → 📭 文档内容为空

---

### Action: update

更新文档内容。新功能，Python 版没有。

**参数：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | string | ✅ | 文档 ID |
| `content` | string | ✅ | 新的 Markdown 内容 |

**行为：**

1. 先读取原文档获取路径
2. 删除原文档
3. 用新内容在相同路径重新创建
4. 返回新文档 ID

**注意：** 思源 API 没有直接的「更新文档」接口，采用「删除+重建」方式实现。文档 ID 会变。

**返回：** 更新后的文档信息和链接

---

### Action: append

追加内容到已有文档。新功能，Python 版没有。

**参数：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | string | ✅ | 文档 ID |
| `content` | string | ✅ | 追加的 Markdown 内容 |

**行为：**

1. 调用 Siyuan `appendBlock` API 在文档末尾追加内容
2. 不读取/重写整个文档，直接通过块操作追加

**返回：** 追加成功确认

**边界：**
- id 或 content 为空 → 错误
- 文档不存在 → 错误

---

### Action: delete

删除文档。对应原 Python 版 `sy-delete`。

**参数：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | string | ✅ | 文档 ID |
| `notebook` | string | 否 | 笔记本序号或名称 |

**行为：**

1. id 不能为空
2. 先 `getDoc` 获取文档路径
3. notebook 为空时走 mapper 默认解析
4. 调用 `removeDoc` 删除

**返回：** 删除确认

**边界：**
- id 为空 → ❌ 文档 ID 不能为空
- 文档路径解析失败 → ❌ 无法确定文档路径
- 删除失败 → ❌ 删除失败：{msg}

---

### Action: move

移动文档。新功能，Python 版没有。

**参数：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | string | ✅ | 要移动的文档 ID |
| `targetNotebook` | string | 否 | 目标笔记本（不指定则用当前笔记本） |
| `targetPath` | string | 否 | 目标路径 |

**行为：**

1. 读取源文档内容和路径
2. 在目标位置用原内容创建新文档
3. 删除源文档
4. 返回新位置信息

**返回：** 移动成功确认 + 新文档链接

**注意：** 思源 API 没有直接的「移动文档」接口，采用「复制+删除」方式实现。

---

### Action: tree

获取文档树结构。对应原 Python 版 `sy-list` 的增强版。

**参数：**

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `notebook` | string | 否 | 索引 0 | 笔记本序号或名称 |
| `path` | string | 否 | `/` | 起始路径 |

**行为：**

1. 调用 `listDocsByPath` 获取指定路径下的文档列表
2. 返回文档标题 + ID 的格式化列表

**返回：**

```
📄 文档列表：
- 我的笔记  `20210101000000-abc1234`
- 项目计划  `20210101000000-def5678`
```

**边界：**
- 笔记本为空 → 返回空列表 📭 该笔记本下没有文档

## 三、sy-search —— 搜索

### 适用场景

用户说「搜索关于 XXX 的内容」「在代码库里找 XXX」「用语义搜索找 XXX」。

### 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `query` | string | ✅ | — | 搜索关键词 |
| `mode` | enum | 否 | `normal` | `normal` 思源全文搜索 / `ai` 语义搜索 / `code` 本地代码搜索 |
| `limit` | number | 否 | 10 | 结果数量上限 |
| `notebook` | string | 否 | — | 限定笔记本（仅 normal/ai 模式） |
| `path` | string | 否 | — | 限定项目名（仅 code 模式） |
| `file_type` | enum | 否 | `code` | 文件类型过滤（仅 code 模式）：`code` / `doc` |
| `context_lines` | number | 否 | 3 | 上下文行数（仅 code 模式） |

### Mode 分布

| Mode | API | 用途 |
|------|-----|------|
| `normal` | `/api/search/searchNotes` | 思源关键词全文搜索 |
| `ai` | `/api/search/searchFullText` | 思源语义/向量搜索 |
| `code` | ripgrep 子进程 | 本地代码库搜索 |

### 返回格式

**normal/ai 模式：**
```
🔍 找到 N 条结果（mode: normal）：
1. 📄 **文档标题**
   > 片段内容...
   📁 /路径
```

**code 模式：**
```
🔍 找到 N 处代码匹配「query」（mode: code）：
1. **repo名称** — `file.ts:42`
   ```
   代码片段
   ```
```

**边界：**
- query 为空 → ❌ 搜索关键词不能为空
- 无结果 → 📭 未找到相关结果 / 📭 代码库中未找到「query」
- code 模式下 rg 未安装 → 提示安装 ripgrep

## 八、通用错误处理

所有工具遵循统一的错误返回格式：

| 错误类型 | 触发条件 | 返回信息 |
|---------|---------|---------|
| 连接错误 | 思源未运行、网络不通 | ❌ 思源笔记未运行，请先启动思源笔记 |
| API 错误 | Siyuan 返回 code !== 0 | ❌ {操作}失败：{msg} |
| 参数错误 | 必填参数为空 | ❌ {参数名}不能为空 |
| mapper 错误 | 笔记本名称找不到 | ❌ 笔记本「{名称}」不存在 |

## 九、工具命名约定

| 规则 | 说明 | 示例 |
|------|------|------|
| 前缀 | 统一 `sy-` | `sy-doc`, `sy-search` |
| 命名 | 英文小写，无缩写歧义 | `sy-notebook` 不是 `sy-nb` |
| 动词 | 用 action 参数而非 tool 名 | `sy-doc action=delete` 不是 `sy-delete` |
| 一致性 | 同一领域用同一 tool | 所有文档操作归 `sy-doc` |
