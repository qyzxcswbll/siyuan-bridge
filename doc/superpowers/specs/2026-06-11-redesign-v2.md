# siyuan-mcp 工具重设计（第二版）

> 日期：2026-06-11
> 状态：待实现

---

## 概述

参考 obsidian-mcp 的低层级设计哲学，将现有工具集从"智能保存"改为**确定性 CRUD**。核心变化：

- 去掉 sy-last/sy-session（伪智能，无法验证内容真实性）
- sy-save 回归简单写入（不预览不确认）
- 新增 sy-read、sy-delete
- sy-list 改为列出文档，sy-notebook 负责笔记本列表

---

## 工具体系

| 工具 | 功能 | 参数 |
|------|------|------|
| `sy-save` | 保存文档（直接写） | content, notebook, path |
| `sy-read` | 读取文档 | path / id |
| `sy-list` | 列出笔记本下的文档列表 | notebook, path |
| `sy-find` | 搜索知识库（不变） | query, mode |
| `sy-delete` | 删除文档 | path / id, notebook |
| `sy-notebook` | 列举笔记本 | — |

---

## 工具详情

### sy-save

**输入参数：**

| 参数 | 必填 | 说明 |
|------|------|------|
| `content` | 是 | Markdown 内容 |
| `notebook` | 否 | 笔记本序号/名称，默认索引0 |
| `path` | 否 | 文档路径，不指定则自动生成 |

**行为：**
- 不预览、不确认、直接写入
- 自动生成路径（含时间戳防覆盖）
- 自动提取标题
- 自动生成标签（可保留也可移除）

**输出：** 文档 ID 和链接

---

### sy-read

**输入参数：**

| 参数 | 必填 | 说明 |
|------|------|------|
| `id` | path/id 二选一 | 文档 ID（siyuan://blocks/xxx） |
| `path` | path/id 二选一 | 文档路径 |

**行为：**
- 调用思源 API 读取文档原始内容

**输出：** Markdown 内容

---

### sy-list

**输入参数：**

| 参数 | 必填 | 说明 |
|------|------|------|
| `notebook` | 否 | 笔记本序号/名称，默认索引0 |
| `path` | 否 | 限定目录路径 |

**行为：**
- 列出指定笔记本下的文档树

**输出：** 文档列表（路径、标题、更新时间）

---

### sy-find

不变，保留现有实现。支持 mode=normal/ai/code。

---

### sy-delete

**输入参数：**

| 参数 | 必填 | 说明 |
|------|------|------|
| `id` | path/id 二选一 | 文档 ID |
| `path` | path/id 二选一 | 文档路径 |
| `notebook` | 否 | 需要和 path 配合 |

---

### sy-notebook

**输入参数：** 无

**行为：**
- 列出所有笔记本（带编号、名称、关闭标记）

**输出：** 编号列表
